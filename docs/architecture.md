# Architecture

bbe-charts is the Helm chart repository for the Brains Beyond Expectations (BBE) platform. It provides all Kubernetes charts consumed by the [bbe-quest](https://github.com/Brains-Beyond-Expectations/bbe-quest) CLI.

---

## Repository Structure

```
bbe-charts/
├── charts/
│   ├── autobrr/          ← Individual application chart
│   ├── bazarr/           ← Individual application chart
│   ├── blocky/           ← Individual application chart
│   ├── prowlarr/         ← Individual application chart
│   ├── radarr/           ← Individual application chart
│   ├── sonarr/           ← Individual application chart
│   ├── bbe-media/        ← Umbrella chart (media stack)
│   └── bbe-networking/   ← Umbrella chart (network stack)
├── library.yaml          ← Autogenerated package registry (consumed by bbe-quest CLI)
├── required-versions.yaml ← Source of truth for chart↔CLI version requirements
└── lib-gen/              ← Go tool that regenerates library.yaml
```

---

## Chart Layers

### Individual Application Charts

Standalone charts for each application. Each chart follows a consistent structure:

- `Deployment` — configurable replicas, image, resources, env vars, probes, security context
- `Service` — ClusterIP (or LoadBalancer for blocky)
- `Ingress` — disabled by default, fully configurable
- `PersistentVolumeClaim(s)` — for config data (and media data where applicable)

See the individual chart docs for details:

- [autobrr](charts/autobrr.md)
- [bazarr](charts/bazarr.md)
- [blocky](charts/blocky.md)
- [prowlarr](charts/prowlarr.md)
- [radarr](charts/radarr.md)
- [sonarr](charts/sonarr.md)

### Umbrella Charts

Umbrella charts bundle multiple applications into a single installable unit via Helm chart dependencies.

- [bbe-media](charts/bbe-media.md) — full media server stack
- [bbe-networking](charts/bbe-networking.md) — network infrastructure

These are the packages exposed to users via `bbe install`.

---

## Integration with bbe-quest

The bbe-quest CLI and bbe-charts interact through `library.yaml`:

```
bbe-quest CLI runtime
       │
       │  HTTP GET
       ▼
library.yaml (this repo, on GitHub)
       │
       │  resolve compatible entry by min-bbe-cli ≤ running CLI version
       ▼
ChartEntry { name, version, repositoryUrl, repositoryName }
       │
       │  helm repo add + helm install/upgrade
       ▼
Kubernetes cluster
```

Only the two umbrella charts (`bbe-media`, `bbe-networking`) are registered in `library.yaml` and therefore installable via the CLI.

---

## Version Compatibility

`library.yaml` contains multiple entries — one per compatibility window. Each entry specifies a `min-bbe-cli` version. At runtime, the CLI selects the entry with the highest `list-revision` that is compatible with its own version.

This allows chart updates to be deployed without requiring all users to upgrade the CLI simultaneously.

See [Library Versioning](library-versioning.md) for details on how this file is managed.
# Architecture

## Overview

bbe-charts is the Helm chart repository for the **Brains Beyond Expectations (BBE)** home-lab platform. It provides two installable umbrella chart bundles — `bbe-media` and `bbe-networking` — built from a collection of individual application charts.

Charts are hosted at `https://brains-beyond-expectations.github.io/bbe-charts` and consumed by the [bbe-quest CLI](https://github.com/Brains-Beyond-Expectations/bbe-quest).

---

## Repository Structure

```
bbe-charts/
├── charts/
│   ├── autobrr/          ← Individual chart
│   ├── bazarr/           ← Individual chart
│   ├── blocky/           ← Individual chart
│   ├── prowlarr/         ← Individual chart
│   ├── radarr/           ← Individual chart
│   ├── sonarr/           ← Individual chart
│   ├── bbe-media/        ← Umbrella chart (bundles media stack)
│   └── bbe-networking/   ← Umbrella chart (bundles network stack)
├── required-versions.yaml  ← Source of truth for chart-to-CLI version requirements
├── library.yaml            ← Autogenerated package registry (consumed by bbe-quest)
└── lib-gen/                ← Go tool that regenerates library.yaml
```

---

## How bbe-quest Consumes This Repository

1. The **bbe-quest CLI** fetches `library.yaml` from this repo at runtime via a direct GitHub raw URL.
2. It finds the **most recent library entry** where `min-bbe-cli <= <running CLI version>`.
3. It extracts the chart list from that entry (chart name, version, Helm repo URL).
4. It runs `helm repo add` + `helm install/upgrade` for each selected package against the user's Kubernetes cluster.

This design means the CLI never bundles chart versions itself — it always resolves them at runtime from `library.yaml`.

---

## Umbrella Charts vs. Individual Charts

| Type | Examples | Purpose |
|---|---|---|
| **Individual** | autobrr, bazarr, blocky, prowlarr, radarr, sonarr | Single-application Helm charts |
| **Umbrella** | bbe-media, bbe-networking | Meta-charts that bundle individual + external charts |

Only the umbrella charts (`bbe-media`, `bbe-networking`) are listed in `library.yaml` and installable via `bbe install`. Individual charts are dependencies of the umbrella charts.
