## Project Overview

bbe-charts is the Helm chart repository for the **Brains Beyond Expectations (BBE)** home-lab platform. It provides pre-configured Kubernetes charts for a complete self-hosted media server and network infrastructure stack.

Charts are published at `https://brains-beyond-expectations.github.io/bbe-charts` and consumed by the [bbe-quest CLI](https://github.com/Brains-Beyond-Expectations/bbe-quest). Users never interact with this repo directly â€” they install packages through `bbe install`.

## Critical Rule: library.yaml is Autogenerated

**Never edit `library.yaml` by hand.** The file starts with:

```
# AUTOGENERATED FILE, DO NOT EDIT!!!
```

Always:
1. Edit chart versions in `charts/<name>/Chart.yaml`
2. Edit CLI version requirements in `required-versions.yaml`
3. Run `cd lib-gen && go run main.go` to regenerate `library.yaml`

## How bbe-quest Consumes This Repo

At runtime, the bbe-quest CLI:
1. HTTP GETs `library.yaml` from this repo via GitHub raw URL
2. Finds the most recent entry where `min-bbe-cli â‰¤ running CLI version` (sorted by `list-revision` descending)
3. Runs `helm repo add + helm install/upgrade` for each chart in that entry

Only the umbrella charts (`bbe-media`, `bbe-networking`) appear in `library.yaml`. Individual charts are internal dependencies.

## library.yaml Structure

```yaml
# AUTOGENERATED FILE, DO NOT EDIT!!!
library:
  - min-bbe-cli: "0.6.1"
    list-revision: 2            # Higher = newer; CLI picks the highest compatible revision
    charts:
      - name: bbe-media
        version: 0.1.1
        repositoryUrl: https://brains-beyond-expectations.github.io/bbe-charts
        repositoryName: bbe
      - name: bbe-networking
        version: 0.1.0
        repositoryUrl: https://brains-beyond-expectations.github.io/bbe-charts
        repositoryName: bbe
```

Entries are sorted descending by `min-bbe-cli`, then descending by `list-revision`. The full version history is preserved â€” old entries are never deleted.

## required-versions.yaml

Declares which umbrella charts are installable and which CLI version they require:

```yaml
charts:
  - name: bbe-media
    min-bbe-cli: 0.6.1
  - name: bbe-networking
    min-bbe-cli: 0.6.0
```

Edit this file when adding a new top-level chart or changing a CLI version constraint.

## lib-gen Algorithm

`lib-gen/main.go` (`processChanges` function):

1. Reads `required-versions.yaml` for chart names and `min-bbe-cli` values
2. For each chart, reads the current version from `charts/<name>/Chart.yaml`
3. Searches existing `library.yaml` entries compatible with `min-bbe-cli`
4. If the current version is **newer** than what's in the compatible entry â†’ creates a new entry with `list-revision` incremented by 1
5. If no compatible entry exists â†’ creates a new entry with `list-revision: 1`
6. Sorts all entries (descending `min-bbe-cli`, then descending `list-revision`)
7. Overwrites `library.yaml`

## Coding Conventions (lib-gen)

- **No third-party dependencies without discussion.** Current deps: `blang/semver`, `spf13/viper`, `stretchr/testify`.
- Tests live in `main_test.go` alongside `main.go`.
- All tests use `github.com/stretchr/testify/assert`.
- `library.yaml` is the only output â€” lib-gen must be idempotent: running it twice on the same input must produce identical output.

## How to Add a New Umbrella Chart

1. Create the chart in `charts/<name>/` with the standard structure above.
2. Add tests in `charts/<name>/tests/`.
3. Add an entry to `required-versions.yaml`.
4. Run `cd lib-gen && go run main.go` to regenerate `library.yaml`.
5. Add documentation at `docs/charts/<name>.md`.
6. Run `helm unittest charts/<name>` to verify tests pass.

## How to Bump a Chart Version

1. Edit `charts/<name>/Chart.yaml` â€” update `version` (and `appVersion` if the app version changed).
2. Run `cd lib-gen && go run main.go` to regenerate `library.yaml`.
3. If this is a sub-chart of an umbrella, also bump the dependency version in the umbrella's `Chart.yaml` and run `helm dependency update charts/<umbrella>`.
